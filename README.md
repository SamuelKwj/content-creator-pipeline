## What is this?

A **self-contained skill pack** that turns any AI agent into a professional content creation workflow engine.

No external dependencies. No platform assumptions. Just copy the folder and your AI knows:

- How to capture inspiration and turn it into scored, tracked ideas
- How to blind-score content against a calibrated rubric (agent isolation вҖ” no self-grading)
- How to write immutable prediction logs before publishing
- How to run post-publish retro analysis and evolve the scoring system

**Works with:** Hermes Agent В· Claude Code В· Codex В· Cline В· any AI that reads markdown skill files.

## How Your AI Agent Uses This

> This section is for **you** (the AI reading this README). Follow these rules.

### On First Load

1. Read `SKILL.md` вҖ” it's the master workflow document
2. If the user hasn't initialized: load `skill/cheat-init/SKILL.md` and guide them through setup
3. Ask: *"What type of content do you create?"* вҖ” never assume

### During a Session

| User says | You load | You do |
|-----------|----------|--------|
| "жҲ‘жңүдёӘжғіжі•" / "зҒөж„ҹ" | `skill/cheat-spark/SKILL.md` | Give 3-4 title options вҶ’ blind score вҶ’ write to inbox |
| "жҲ‘иҰҒеҸЈиҝ°" / "жҲ‘иҜҙдҪ и®°" | `skill/cheat-spark/SKILL.md` | Enter dictation mode: transcribe only, don't archive |
| "жҲ‘жғіеҒҡдёҖжқЎе…ідәҺXзҡ„" | `skill/cheat-seed/SKILL.md` | Deep-dive conversation вҶ’ write draft outline |
| "жү“еҲҶиҝҷзҜҮ [path]" | `skill/cheat-score/SKILL.md` | Delegate to blind sub-agent вҶ’ display scores |
| "еҗҜеҠЁйў„жөӢ [path]" | `skill/cheat-predict/SKILL.md` | Full prediction flow вҶ’ immutable log |
| "жӢҚдәҶ [path]" | `skill/cheat-shoot/SKILL.md` | Create video folder, register shoot |
| "е·ІеҸ‘еёғ [url]" | `skill/cheat-publish/SKILL.md` | Write URL + timestamp to prediction header |
| "еӨҚзӣҳ [path]" | `skill/cheat-retro/SKILL.md` | Collect data вҶ’ compare prediction vs reality |
| "зҠ¶жҖҒ" / "зңӢжқҝ" | `skill/cheat-status/SKILL.md` | Show calibration progress, buffer, pending retros |
| "еҚҮзә§ rubric" | `skill/cheat-bump/SKILL.md` | Re-score calibration pool вҶ’ evolve rubric |
| "дјҳеҢ–ејҖеӨҙ" | `skill/dbs-hook/SKILL.md` | Diagnose hook вҶ’ generate alternatives |
| "еҺ» AI е‘і" | `skill/humanizer/SKILL.md` | Strip AI tells, restore human voice |

### Critical Rules (DO NOT VIOLATE)

1. **Blind scoring only.** Never inline-score content. Always delegate to `cheat-score-blind` sub-agent.
2. **Predictions are immutable.** Once written, the prediction block cannot be changed. Retros append separately.
3. **Don't write the full script.** AI provides outlines and skeletons. The creator writes the final draft.
4. **Title options before archiving.** Never save a spark without offering 3-4 title angles first.
5. **Don't assume content type.** Always ask what kind of content the user creates.

## Philosophy

> **"еҺҢи ўжҳҜиөҡй’ұзҡ„ж•Ңдәә вҖ” е…ҲеҸ‘еҶҚдјҳеҢ–"**
> *Hating imperfection is the enemy of making money. Ship first, optimize later.*

This pipeline is built on a simple insight: most creators fail not because their content is bad, but because they have no **calibration loop**. They ship, they feel, they guess.

This toolkit replaces guessing with a **scoring вҶ’ prediction вҶ’ retro вҶ’ evolve** cycle. Every piece of content becomes a data point. Every data point sharpens the rubric. Every sharpened rubric makes the next prediction better.

It's not about being perfect. It's about getting better, measurably, every time.
